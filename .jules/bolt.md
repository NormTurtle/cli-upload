## 2024-03-26 - Buffer global progress updates to reduce lock contention
**Learning:** In highly concurrent upload scripts (like `uc.py` which uses ThreadPoolExecutor for multiple file uploads), tracking progress by calling a synchronized global method (`add_progress`) on every small chunk read (e.g., inside `_MultipartStream.read()`) creates severe lock contention. It essentially forces threads to serialize for updating a global counter, degrading the benefits of multi-threading. Also, when implementing retryable buffered streams, rollback logic within `seek()` must accurately subtract any un-flushed local buffer from the global calculation to prevent negative overall progress.
**Action:** When working on threaded streams or downloads/uploads in Python, always buffer global progress tracking updates locally within the object/thread, and only flush them to the global synchronized block when a reasonable threshold (e.g., 512 KB) or EOF is reached. Always ensure rollback logic accurately accounts for this local state.
